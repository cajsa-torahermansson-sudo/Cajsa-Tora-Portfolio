Usability testingUXUIAccessibilityDesign system

Wellbeing and Taste cross-functional teams — cleaning and kitchen appliances. Three major projects including a robot vacuum UI and the hood light picker redesign. One of the few non-NDA projects I can share publicly.

Problem
Usability study with 8 target users revealed confusion: non-clickable elements treated as interactive, icons too small, layout with too much empty space. The automated schedule was universally misread as customisable — creating distrust and unclear interactions throughout.
Process
4-step iterative cycle: guerrilla testing + dot voting on brightness perception → A/B test slider vs circular picker → removed friction-causing Save button → WCAG AA contrast improvements in collaboration with the Shape design system team.
Outcome
WCAG AA contrast achieved. Clearer selection marker. Reduced UX copy. Redesigned auto schedule that clearly communicates the lighting logic. Fan speed quick-win — removing the Save button appreciated globally by the design team.
Tools
FigmaMirrorFigJamTestFlightGitHubJIRAConfluenceGuerrilla testingUsability tests
Key learning
The colour slider — most preferred in testing — couldn't ship due to technical constraints discovered late. Lesson: sync with dev before you've run four rounds of testing on something that can't be built yet.
Service designLean UXUIAccessibilityUsability testing

SMS Livräddaren (SE) / Trygfonden (DK) — CPR volunteer dispatch app, a decade old and overdue for redesign. First in-house project at Qvik AB. I joined a month in with no clear requirements documented, a tight deadline, and a team of three developers.

Problem
Alarm notifications covered the full screen — blocking the map at the exact moment users needed it to locate a patient. Confusing navigation, poor copy, broken icon logic. The core interaction was actively working against users in emergency conditions.
Process
Ran Scope Canvas and Service Blueprint workshops to align team and client from scratch. Drafted roadmap. Applied "Don't Make Me Think" with map as the central element. Validated with 8 SMS Livräddare users — confirmed improved direction, flagged UX copy refinements needed.
Constraints
Reskinning model — reusing existing source code. Client unfamiliar with agile, continuously pushing new features after MVP scope was locked. Late-stage design approvals. Dev team working with delayed inputs throughout.
Tools
FigmaMiroJIRAConfluenceGitHubService BlueprintScope Canvas
Key learning
Not my proudest project — but one of the most instructive. Discovered The Dangerous Animals Handler's Toolbox for navigating difficult project dynamics. Real project management experience: how to protect the team, maintain scope, and keep a client aligned when agile isn't in their vocabulary.
UXUIRapid prototypingDual TrackIoTUsability testing

New app for Bluetooth-connected 300-series garden tools — notifications, usage tracking, digital support guides. End-to-end over two years as the team scaled from 6 to 15. Front-end in Jönköping, QA in Mumbai, design in Stockholm.

Problem
First MVP tested in Stockholm and Charlotte, NC returned largely negative results — poor usability, difficult navigation, core features impossible to find. The team made the call to start over rather than patch a broken foundation.
Process
Adopted Dual Track agile. New features reviewed across design, backend, frontend, and product leads in 3-week sprints before entering the design backlog. I rotated between Stockholm and Jönköping every two weeks — the first designer to do this — which transformed dev collaboration.
Outcome
Second usability test with 8 users validated core features: device pairing, spare parts lookup, dealer search. MVP shipped and iterated for a year post-launch. Also contributed to Husqvarna's first design library, Fleet service, a chainsaw blade POC, and an auto-mower hackathon.
Tools
SketchZeplinJIRAConfluenceDual TrackRapid prototyping
Key learning
Physical proximity to devs matters more than any process framework. Rotating workstations built trust and resolved problems before they became tickets. Dual Track took time to learn but was the right tool for managing complexity at this scale.
ResearchStrategyUXUIWorkshop facilitation

Savings & Investment web division. Two parallel tracks: redesigning the financial overview as a responsive balance sheet, and launching a new savings goal service — both aimed at reversing declining engagement with digital savings products.

Problem
Customers defaulted to branch visits over digital — partly poor UX, partly CMS constraints limiting what could be built. Declining interest in ISK accounts and funds needed a product response, not a marketing one.
Process
Competitive analysis, Google Analytics deep dive, persona workshop, innovation sprint with Crazy 8s and low-hanging fruit prioritisation. A/B testing confirmed preference for the streamlined savings goal flow. Priority mapping evaluated ideas on usability value vs implementation effort.
Outcome
Financial overview MVP with responsive balance sheet and property asset workflow. Savings goal service aligned to the design system. Internal tech test with 100+ employees. Weekly cross-platform syncs with the app team to maintain consistency throughout.
Tools
FigmaMiroSAFeDouble DiamondJIRAConfluenceGoogle Analytics
Key learning
In large legacy organisations, the CMS is often the real constraint — not stakeholders, not budget. Understanding technical boundaries early shapes what good design actually means in that context.
User interviewsUser storiesUXPublic sectorQuantitative testing

Digitala samarbeten — Skatteverket's API and open data platform — suffered from low traffic and poor engagement. My role: interview active users across roles, synthesise into user stories, and design an MVP compatible with the current CMS while preparing for the KIL:en migration.

Problem
Low traffic driven by navigational friction, lack of content, and inaccessible APIs. Users — from developers to decision-makers — couldn't find what they needed or understand what was available.
Process
Interviews with companies using the APIs across roles (developers, strategists, decision-makers). Three user stories emerged: generating interest via referrals, staying updated on new APIs, and resolving API issues. Led collaborative sketching session then rapid wireframe prototype for testing and demos.
Outcome
Internal technical test with UX, copywriters, and technical specialists. Quantitative survey sent to ~30 companies using the service — 10 participated. Feedback confirmed MVP readiness. Key insight: for 20 survey participants, budget a contact list of at least 100.
Tools
FigmaMiroSAFeSitevisionJIRAConfluenceGoogle Analytics
Context
Remote-first during COVID-19, and expecting my son Levi. A reminder that good work gets done regardless of circumstances — and that some of the most focused design thinking happens when the world is chaotic around you.

The Turbo+™ Insights device monitors inhalation and medication adherence via Bluetooth, surfacing data in a companion mobile app with pre-configured reminders. My role: design conceptual solutions for upcoming features and hypotheses within a structured software lifecycle.

Context
MedTech regulatory environment shapes every design decision. This is not casual software — it monitors medication adherence for patients with respiratory conditions. Clarity, trust, and accuracy are non-negotiable from the start, not a later phase.
Approach
Lean UX within a structured software lifecycle. Designed and validated conceptual solutions for new feature hypotheses using A/B testing and usability testing before development investment. Worked within strict regulatory and accessibility requirements throughout.
